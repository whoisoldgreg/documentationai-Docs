<Update label="2024-10-10" description="v2.0.0" tags={["feature", "breaking"]}>

## New Features

- Launched flexible recurring billing with automated invoicing, dunning sequences, and custom billing frequencies.
- Added support for subscription pausing, prorated refunds, and webhook notifications for billing events.

## Breaking Changes

- Updated API endpoints for billing: `/v1/billing` now requires `{merchantId}` path parameter.
- Deprecated legacy recurring endpoints; migrate using the guide below.

<CodeGroup tabs="JavaScript,cURL">
````javascript
// New endpoint usage
const response = await fetch('https://api.example.com/v2/billing/{merchantId}/subscriptions', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${YOUR_TOKEN}` },
  body: JSON.stringify({ plan: 'monthly', amount: 99.99 })
});
````
````bash
curl -X POST "https://api.example.com/v2/billing/{merchantId}/subscriptions" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"plan": "monthly", "amount": 99.99}'
````
</CodeGroup>

## Bug Fixes

- Corrected token refresh logic in SDKs during long-running billing operations.

</Update>
